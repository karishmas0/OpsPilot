from rank_bm25 import BM25Okapi
from typing import List, Tuple
import re

def tokenize(text: str) -> List[str]:
    return re.findall(r"[A-Za-z0-9_:/.-]+", text.lower())

class BM25Index:
    def __init__(self, texts: List[str], ids: List[str]):
        self.ids = ids
        self.tokens = [tokenize(t) for t in texts]
        self.bm25 = BM25Okapi(self.tokens)

    def search(self, query: str, top_k: int = 6) -> List[Tuple[str, float]]
        scores = self.bm25.get_scores(tokenize(query))
        pairs = list(zip(self.ids, scores))
        pairs.sort(key=lambda x: x[1], reverse=True)
        return pairs[:top_k]

43. “Interview trade-offs pack” (crisp senior answers)
Use these as rehearsed mini-speeches.

1. Why hybrid retrieval?
Vector search captures semantic similarity; BM25 nails exact alert names, metric

labels, and error codes. Hybrid improves Recall@K especially on short queries.

 120/180



2/7/26, 6:59 PM AI project blueprint

2. Why FAISS local instead of managed vector DB?
Constraint: free + laptop-friendly. FAISS gives production-like performance without

external dependencies. I abstracted vector store so swapping to Qdrant is trivial

later.

3. How do you prevent hallucinations?
Hard requirement: every action must cite evidence doc_ids from retrieved context. I

enforce this in code ( validate_grounded_actions ) so it’s not just “prompting”.

4. Why use an agent at all?
Because the workflow is multi-step and tool-based: anomaly scoring, retrieval,

structured synthesis, postmortem drafting. LangGraph provides a deterministic

state machine and debuggable trace.

5. How do you evaluate GenAI without paid judges?
I use retrieval metrics (Recall@K, MRR), groundedness checks (% actions with

evidence), and a human feedback UI. This is realistic for teams without paid eval

infra.

6. CPU-only model choice trade-off
Small quantized LLM reduces latency and cost; I compensate by emphasizing

retrieval grounding and structured templates. The LLM is used for synthesis, not

“knowledge”.

7. Why DVC + MLflow together?
DVC versions data and pipelines; MLflow tracks experiments/metrics and acts as a

registry. Together they provide end-to-end reproducibility.

8. What about prompt injection from runbooks?
Treat runbook text as untrusted. Strip suspicious patterns, never execute

commands, and ensure recommendations are allow-listed and evidence-bound.

 121/180



2/7/26, 6:59 PM AI project blueprint

9. How do you handle drift?
Monitor template distribution and anomaly score distribution. If drift exceeds

thresholds, trigger retraining pipeline; keep old model as fallback.

10. How would you scale this?
Async ingestion + reindexing, Redis cache, background job queue, and a dedicated

vector DB. Horizontally scale API; keep indexes versioned and hot-swappable.
